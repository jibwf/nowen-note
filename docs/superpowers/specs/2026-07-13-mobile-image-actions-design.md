# 移动端图片操作菜单设计

## 目标

修复图片节点在手机端被选中后操作菜单不可见的问题，并将现有操作整理为适合触摸操作的渐进式菜单。Web 端悬浮工具条保持不变。

## 交互设计

- 点击图片后保持图片节点选中，收起软键盘并显示底部紧凑操作栏。
- 一级操作为：查看大图、下载、替换、编辑、更多。
- “更多”展开二级面板，提供尺寸、旋转/翻转、复制地址和删除操作。
- 点击遮罩、关闭按钮或系统返回键关闭菜单；编辑器因收起键盘而失焦时，只要图片节点仍被选中，就不关闭菜单。
- 底部菜单适配安全区，一级按钮触控高度不小于 44px，二级面板最大高度不超过 `min(42dvh, 360px)`。
- 删除操作使用危险色，其余操作保持现有主题色与组件风格。

## 实现边界

- 复用 Tiptap 现有图片操作函数和 `ImageExperienceBridge`，不复制上传、替换、下载或持久化逻辑。
- 修复 `blur` 与图片菜单打开之间的竞态，确保收起 Android 键盘不会反向关闭菜单。
- 桌面端图片工具条、表格菜单和分享页大图预览不改变。

## 验收标准

1. 手机端点击图片后能稳定看到底部操作栏，不会闪现后消失。
2. 一级操作与“更多”中的二级操作均可触发原有行为。
3. 点击遮罩、关闭按钮或返回键可关闭菜单。
4. Web 端仍显示原悬浮工具条。
5. 图片菜单识别、失焦处理和移动端展示条件均有回归测试。
