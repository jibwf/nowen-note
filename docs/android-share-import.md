# Android 系统分享与“用 Nowen 打开”

Nowen Note Android 客户端可以作为系统分享目标，直接接收浏览器、微信、QQ、相册和文件管理器发送的内容。

## 支持入口

Android Manifest 注册了以下入口：

- `ACTION_SEND`：单张图片、单个文件、网页标题、链接和选中文本；
- `ACTION_SEND_MULTIPLE`：多张图片或多个文件；
- `ACTION_VIEW`：文件管理器中的“用其他应用打开”。

客户端不会注册普通 `http/https` 网页跳转，因此不会抢占系统默认浏览器。网页应通过系统“分享”进入 Nowen。

## 导入方式

收到分享后，应用底部会打开轻量导入面板。用户可以选择：

1. **保存到文件管理**
   - 可选个人空间或工作区；
   - 可选附件目录；
   - 分享文字不会被静默丢弃，会继续留在待处理队列。
2. **插入已有笔记**
   - 选择空间、笔记本和具体笔记；
   - 文本、链接、图片和附件追加到正文末尾；
   - 保持目标笔记原有 Markdown、HTML 或 Tiptap JSON 格式。
3. **新建笔记并插入**
   - 可编辑自动生成的标题；
   - 标题优先取网页标题，其次取正文首行、文件名或网站域名。

上传成功后会触发打开目标笔记事件。单个文件失败不影响其他文件，失败项继续保留，可再次重试。

## content URI 与临时文件

外部应用通常只授予一次性的 `content://` 读取权限。Nowen 在收到 Intent 后会立即：

1. 在后台读取来源流；
2. 将文件复制到应用私有目录：

   ```text
   files/pending-share-imports/<payloadId>/<itemId>.<ext>
   ```

3. 只持久化私有文件名和安全元数据，不保存来源 URI，也不依赖“真实磁盘路径”；
4. 在上传完成、用户忽略或主动放弃后清理对应文件。

待处理队列保存在应用私有 `SharedPreferences` 中，因此分享发生在未登录状态、Activity 重建或应用重启后，登录成功仍可继续处理。

自动清理策略：

- 最多保留 20 组分享；
- 每组最多 100 个文件；
- 待处理文件总量最多 2GB；
- 超过 7 天的临时内容自动清理。

## 大文件上传

原生插件使用 `HttpURLConnection` + 固定长度 multipart 流式上传，不会把完整文件转成 Base64 或一次性读入 WebView 内存。

- 单文件最大 1GB，与 `/api/files/upload` 上限一致；
- 每 64KB 流式读取；
- 前端显示实时百分比；
- 当前文件可取消；
- 服务器失败时临时文件和队列仍保留，允许重试。

## 安全边界

来源应用提供的文件名、MIME 和字节均视为不可信。

客户端会执行：

- 清理 `/`、`\\`、控制字符、前导点和超长文件名；
- 阻止 APK、EXE、MSI、脚本、动态库、快捷方式等危险扩展名；
- 阻止危险 MIME；
- 检查 `MZ`、ELF、Java Class 和 shebang 等可执行文件头；
- 对 PDF、PNG、JPEG、GIF、WebP、BMP、ZIP、DOCX、XLSX、PPTX、RAR、7z 做文件头识别；
- MIME 与实际内容不一致时使用识别后的类型，并在 UI 标注；
- 私有文件读取时再次做 canonical path 边界检查；
- 上传接口继续执行服务端鉴权、工作区权限和文件类型限制。

附件写回笔记时只保存 `/api/attachments/<id>` 相对 URL，不写死服务器域名或 Electron 动态端口。

## 人工验收清单

建议至少覆盖：

1. Chrome 分享网页标题、链接和选中文本；
2. 微信或 QQ 分享单张图片；
3. 相册多选图片分享；
4. 文件管理器用 Nowen 打开 PDF；
5. 分享 DOCX、ZIP 和未知 MIME 文件；
6. 分享 APK、伪装成图片的 EXE、带 `../` 的文件名；
7. 未登录时分享，随后登录恢复导入；
8. 上传过程中杀掉 Activity，再次打开查看待处理队列；
9. 多文件中一个失败，确认其他文件成功且失败项可重试；
10. 分别导入 Markdown、HTML、富文本笔记，确认旧正文不丢失；
11. 个人空间和工作区权限不足场景；
12. 完成、忽略和放弃后检查私有临时文件已清理。
