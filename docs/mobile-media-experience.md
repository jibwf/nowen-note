# 移动端图片与视频插入、预览和播放

Issue #214 的目标不是单独增加一个“选择视频”按钮，而是补齐移动端媒体从选择、上传、写入正文，到刷新后预览、播放、替换和下载的完整闭环。

## 图片与视频选择

移动端点击编辑器的图片或视频按钮时，会打开统一的轻量底部面板：

- **系统相册**：允许同时选择多张图片或多个视频；
- **拍照 / 拍视频**：根据用户点击的媒体入口调用后置相机；
- **选择文件**：从系统文件管理器选择图片或视频，支持多选。

选择完成后不会立即上传，而是先展示预检列表：

- 文件名；
- 图片或视频类型；
- 单文件大小与总大小；
- 本地缩略预览；
- 空文件、不支持格式、超过 1GB 等前置错误；
- 超过 100MB 的大文件耗时提示。

一次最多处理 30 个媒体文件。图片和视频继续复用编辑器现有的 drop 上传链路，因此 Markdown 与富文本编辑器的最终写入格式保持一致。

## 批量上传反馈

统一上传生命周期事件覆盖：

- 等待上传；
- 正在上传；
- 上传成功；
- 上传失败；
- 失败项单独重试。

多个文件顺序处理时，单个失败不会让已成功的文件回滚，也不会阻断后续文件。用户关闭面板时，已经交给编辑器的上传继续在后台执行。

媒体面板打开期间会临时把编辑器 `inputmode` 设置为 `none`，并主动关闭 Capacitor Keyboard，避免选择相册、操作媒体时正文键盘反复弹出。

## 图片正文体验

图片继续使用项目已有的 `ImageExperienceBridge`：

- 正文内联显示；
- 点击查看大图；
- 旋转；
- 下载；
- 复制；
- 替换；
- 删除；
- 小、中、大、自适应尺寸；
- 移动端底部操作面板。

本次统一媒体入口不会改变已有图片节点格式，刷新、同步和导出行为保持兼容。

## 视频正文体验

本地附件视频在正文中获得增强覆盖层：

- 首帧海报图；
- 时长标记；
- 居中的播放按钮；
- 原生暂停、进度和音量控制；
- 横屏、竖屏比例自适应；
- 双击或按钮全屏；
- 加载失败原因提示与重新加载；
- 移动端长按或“操作”按钮打开底部面板。

视频操作面板包括：

- 播放 / 暂停；
- 全屏；
- 替换；
- 下载；
- 复制地址；
- 删除。

替换视频采用“先上传并插入新视频，确认新节点出现后再删除旧节点”的顺序。上传失败时旧视频保持不变，避免因为替换失败导致正文数据丢失。

## 视频 MIME 兼容

部分 Android 文件提供器会给 MP4 返回空 MIME 或 `application/octet-stream`。附件上传成功后，服务端只针对已知视频扩展名补全 MIME：

| 扩展名 | Content-Type |
| --- | --- |
| mp4 / m4v | video/mp4 |
| webm | video/webm |
| ogv / ogg | video/ogg |
| mov / qt | video/quicktime |
| 3gp / 3g2 | video/3gpp / video/3gpp2 |
| avi | video/x-msvideo |
| mkv | video/x-matroska |

已有明确 MIME 的上传不会被覆盖，高危 MIME 和原有安全限制仍由附件核心路由负责。

## HTTP Range 与拖动播放

附件视频和音频现在支持单段 `Range: bytes=...` 请求：

- 返回 `206 Partial Content`；
- 返回 `Content-Range`、`Content-Length` 和 `Accept-Ranges: bytes`；
- 支持 `start-end`、`start-`、`-suffix`；
- 不支持多段 range，请求返回 `416`；
- `download=1` 继续使用完整强制下载；
- Range 响应标记 `Content-Encoding: identity` 与 `no-transform`，防止 gzip 改变字节偏移；
- 本地存储通过受限文件读取，S3 通过原生 Range GET，不再为了拖动进度条读取完整视频。

Range 下载沿用原附件下载的权限模型：

- 已登录上下文中的笔记 ACL；
- 附件签名 URL；
- 兼容配置允许时的旧 UUID 链接。

## 自动化测试

后端测试覆盖：

- 有界、开放结尾和后缀 Range；
- 多段、非法和超界 Range；
- 视频 `206` 响应内容；
- `416 Content-Range`；
- `download=1` 强制下载；
- Android 空 MIME 的 MP4 归一化。

前端测试覆盖：

- 图片和视频 MIME / 扩展名识别；
- 空文件和不支持文件的上传前校验；
- 时长格式；
- 下载 URL 保留签名参数并移除 `inline=1`。

## 真机验收建议

1. Android 与 iOS 分别从相册多选图片和视频；
2. 分别调用拍照和拍视频；
3. 从文件管理器选择 MP4、MOV、WebM 以及空 MIME 的 MP4；
4. 混合选择图片和视频，检查逐项成功、失败和重试；
5. 在 Markdown 与富文本笔记中插入，刷新后确认节点仍存在；
6. 播放 MP4，拖动进度、暂停、继续和全屏；
7. 竖屏与横屏视频检查比例和页面宽度；
8. 断网制造播放错误，检查重试占位；
9. 替换视频时制造上传失败，确认旧视频不被删除；
10. 长按视频，验证下载、复制和删除；
11. 打开媒体面板时确认正文键盘不弹出；
12. 使用签名附件 URL 和工作区只读账号验证权限行为。
